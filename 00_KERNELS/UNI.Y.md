# UNI.Y

Universal Meaning and Governance Doctrine Kernel

Status: SEED
Layer: Kernel
Domain: Meaning

## Purpose

UNI.Y defines meaning, doctrine, conceptual boundaries, accepted terms, and governance interpretation.

UNI.Y answers what a term means before UNI.Z enforces behavior.

## Core Rule

Definitions must be explicit before they are enforced.

If meaning is unclear, the object or term must be marked NEEDS_CLARIFICATION before routing, validation, publication, or preservation.

## Language Boundaries

- Canon means internal virtual definition state.
- Dynamic Structure means active folder-system behavior.
- Genuine State means verified or authenticated object condition.
- Registry means structured record layer.
- Library means knowledge and preservation layer.
- Storage means location, not authority.

## Canon Boundary

Canon is valid only as the virtual state of a definition.

Canon is not the active folder-system name.

Because Canon is also a major external brand/company name, avoid using Canon as a public-facing folder, product, service, or system-layer name unless reviewed.

## Primary States

- MEANING_DEFINED
- MEANING_UNCLEAR
- TERM_APPROVED
- TERM_BLOCKED
- LEGACY_LANGUAGE
- NEEDS_CLARIFICATION

## Boundary

UNI.Y preserves meaning without confusing language with execution.
