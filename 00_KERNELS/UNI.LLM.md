# UNI.LLM

Universal Language Model Interpretation Kernel

Status: SEED
Layer: Kernel
Domain: AI Interpretation

## Purpose

UNI.LLM governs how AI systems, agents, assistants, and language models interpret, classify, validate, route, and respond inside the operating system.

## Core Rule

No AI system may flatten authority, ownership, storage, brand identity, service identity, workspace function, bridge function, canon state, dynamic structure, or genuine-state status into the same concept.

## Required Behavior

The AI must preserve:

- authority boundaries
- ownership boundaries
- relationship maps
- source proof
- storage roles
- brand and service distinction
- laboratory and library distinction
- canon definition state
- dynamic structure behavior
- genuine-state status
- time continuity

## Primary States

- LLM_INTERPRETED
- LLM_UNCERTAIN
- LLM_NEEDS_CLARIFICATION
- LLM_VERIFIED
- LLM_UNVERIFIED
- LLM_ROUTED_TO_LAB
- LLM_ROUTED_TO_LIB
- LLM_BLOCKED
- LLM_REFUSED_MERGE
- LLM_READY_FOR_HUMAN_REVIEW

## Boundary

UNI.LLM protects interpretation without pretending interpretation is truth.
