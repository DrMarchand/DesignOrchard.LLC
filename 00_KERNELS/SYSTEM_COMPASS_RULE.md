# System Compass Rule

Atlas is the map. The System Compass is the directional decision layer.

The System Compass does not replace Atlas. It bounces off Atlas.

Atlas answers:
- where the object is
- what the object connects to
- what routes exist

The System Compass answers:
- which direction the system should move next
- which boundary is most important
- whether the object should build, preserve, review, or protect

## Compass Points

North: Authority, ownership, protected root, LLC boundary.
East: Execution, workspace movement, build flow, active routing.
South: Preservation, Library, Vault, proof, archive, restore source.
West: Reflection, Notebook, Journal, review, lessons, correction.
Center: System ID, object identity, stable reference point.

## Rule

No object should be promoted, copied, archived, vaulted, or connected unless Atlas can map it and the System Compass can justify the direction of movement.

## Boundary

Atlas is topology.
System Compass is directional intent.
System ID is stable object identity.
Vault is protected master preservation.
