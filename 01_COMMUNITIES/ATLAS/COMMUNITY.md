## Definition

DrMarchand Atlas is the shared operating map, topology layer, and coordination surface used by the operating system and the execution engine.

Atlas maps the ecosystem so the operating system can govern it and the engine can execute it.

## Placement

Atlas exists inside the operating system, inside the engine, and between them as the shared map layer.

## Mirror Rule

Atlas is the living workspace.
GitHub is the durable technical mirror.
Asana is the execution mirror.
Google Drive is the live workspace document mirror.

## Structural Model

Ecosystem -> Community -> Home -> Package or Node or Record or Service

## Connection Record Rule

A connection should live in three places:

1. Atlas: the map route showing what the connection links.
2. Notebook: the working explanation, tests, sketches, and active design notes for the connection.
3. Journal: the chronological record of creation, change, validation, breakage, repair, promotion, or retirement.

Atlas shows where the connection is.
Notebook explains how the connection works.
Journal records what happened to the connection over time.

A connection is not fully validated unless its Atlas route, Notebook logic, and Journal history can be reconciled.

## Boundaries

Atlas is not only documentation.
Atlas is not only a folder.
Atlas is not only GitHub.
Atlas is not only Asana.
Atlas is the map layer.
