# Laboratory Engine and OS Domain Routing Rule

drmarchandslaboratory.com is the Core domain from which the Engine organizes Laboratory activity, system structure, runtime logic, validation routes, and workspace-derived signals.

os.drmarchandslaboratory.com is the Home for DrMarchand's OS.

It is the operating-system surface where governance, system rules, navigation, schemas, and execution relationships can be presented and coordinated.

## Domain Roles

- drmarchandslaboratory.com = Core / Engine organizing root
- os.drmarchandslaboratory.com = Home for DrMarchand's OS
- Engine = organizes from the Core
- OS = governs and coordinates through its Home

## Boundary

The Core is not the same thing as the OS Home.

The Engine may organize from the Core, while the OS Home presents and coordinates the operating framework.

## Routing

Core activity flows through the Engine.

OS-level rules and governance surface through os.drmarchandslaboratory.com.

Library-facing resolved data may route toward DrMarchandsLibrary.org when it becomes organized record material.
