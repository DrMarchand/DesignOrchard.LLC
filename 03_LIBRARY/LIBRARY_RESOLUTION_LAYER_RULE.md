## Core Rule

The Library is not a Workspace.

A Workspace is where work is performed, built, tested, designed, produced, or managed.
The Library is a resolved observation and preservation layer derived from Workspace activity.

## Purpose

The Library lets the system see what happened in a Workspace without becoming the Workspace and without taking every raw datum ever created by that Workspace.

## Distinctions

Workspace = active work environment.
Library = resolved record layer.
Archive = preserved long-term storage.
Notebook = active reasoning and working notes.
Journal = chronological event record.

## Input Rule

The Library receives selected, resolved, validated, or meaningful outputs from a Workspace.

The Library does not automatically ingest every draft, temporary file, raw asset, failed experiment, or transient working state.

## Validation

Before an object enters the Library, Atlas should determine whether it is resolved, meaningful, proof-backed, or intentionally preserved.

If it is only raw Workspace activity, it should remain in the Workspace, Notebook, Journal, or temporary working layer until resolved.
