## Source of truth rule

Atlas is the living workspace where the operating structure is actively shaped, reviewed, and evolved.

GitHub is the durable technical mirror for versioned files, manifests, kernels, schemas, and change history.

Asana is the execution mirror for tasks, review items, cleanup work, and operational follow-through.

The intended flow is:

```text
Atlas -> GitHub -> Asana
```

Atlas-first does not mean GitHub is secondary in importance. It means GitHub records the system after Atlas produces or clarifies the working structure.

## Purpose

This manifest initializes the kernel, community, home, connection, and package layer for the operating structure.

The root OS structure is moving from passive folders into package-aware system communities and homes.

## Core language boundary

Canon remains valid as an internal virtual definition state.

Canon must not be used as a public-facing folder, product, service, or system-layer name unless separately reviewed.

Dynamic Structure is the active folder-system behavior layer.

Genuine State is the verified/authenticated condition of an object, record, package, or structure.

## Community and home rule

Homes exist inside Communities.

Communities exist inside the Ecosystem.

Storage may attach to Homes, but storage is not automatically a Home.

A Home without a Community is only an address.

A Home inside a Community has meaning, responsibility, and relationship context.

## Definitions

### Atlas

Living workspace and primary operating surface for shaping the ecosystem.

### GitHub

Durable technical mirror for versioned source, manifests, kernels, and auditable change history.

### Asana

Execution mirror for tasks, cleanup work, operational reviews, and follow-through.

### Ecosystem

The full Design Orchard LLC operating world.

### Communities

Domain neighborhoods with shared purpose, rules, roles, and relationships.

### Homes

Stable places inside a Community where packages, agents, records, divisions, services, or connections may live.

### Connections

Bridge paths between communities, homes, platforms, agents, storage providers, registries, and workflows.

### Packages

Portable bundles of files, rules, metadata, assets, or logic that can move without losing identity.

### Dynamic Structure

Live folder behavior, routing behavior, structure testing, promotion logic, and gap detection.

### Genuine State

Verified/authenticated condition of an object, record, package, or structure.

### Canon

Internal virtual state of the definition. Canon may define meaning, but it does not describe live folder behavior.

## Validation questions

Every community/home/package must eventually answer:

1. What community does this belong to?
2. What is this home?
3. What authority owns it?
4. What kernel governs it?
5. What packages can live here?
6. What connections can leave here?
7. What gets promoted from here?
8. What gets blocked here?
9. What proves its Genuine State?
10. What gets mirrored to GitHub?
11. What becomes an Asana execution item?

## Execution principle

Do not add complexity before defining the community, home, kernel, package, and connection boundaries.

Structure must remain testable, routable, reversible, and mirrorable.
