## Purpose

This manifest initializes the kernel, home, connection, and package layer for the operating structure.

The root OS structure is moving from passive folders into package-aware system homes.

## Core language boundary

Canon remains valid as an internal virtual definition state.

Canon must not be used as a public-facing folder, product, service, or system-layer name unless separately reviewed.

Dynamic Structure is the active folder-system behavior layer.

Genuine State is the verified/authenticated condition of an object, record, package, or structure.

## Definitions

### Kernels
Core logic seeds, rules, protocols, schemas, and executable system definitions.

### Homes
Stable system addresses where packages, agents, records, divisions, or connections live.

### Connections
Bridge paths between homes, platforms, agents, storage providers, registries, and workflows.

### Packages
Portable bundles of files, rules, metadata, assets, or logic that can move without losing identity.

### Dynamic Structure
Live folder behavior, routing behavior, structure testing, promotion logic, and gap detection.

### Genuine State
Verified/authenticated condition of an object, record, package, or structure.

### Canon
Internal virtual state of the definition. Canon may define meaning, but it does not describe live folder behavior.

## Kernel candidates

- UNI.X: horizontal compass, dynamics, routing, plane orientation
- UNI.Y: vertical governance and movement

- UNI.LAB: laboratory operations
- UNI.LIB: library records and preservation
- UNI.LLM: AI interpretation and routing behavior
- UNI.A: authority boundary module

## Validation questions

Every home/package must eventually answer:

1. What is this home?
2. What authority owns it?
3. What kernel governs it?
4. What packages can live here?
5. What connections can leave here?
6. What gets promoted from here?
7. What gets blocked here?
8. What proves its Genuine State?

## Execution principle

Do not add complexity before defining the home, kernel, package, and connection boundaries.

Structure must remain testable, routable, and reversible.
